# Contribution Guidelines

Thank you for contributing to Awesome AI Accountability.

## Adding an entry

1. Ensure your suggestion is within the scope of AI accountability: frameworks, regulations, tools, papers, organizations, datasets, or media resources.
2. Submit a pull request adding the entry to the bottom of the relevant category.
3. Use the following format:

```
- [Name](URL) - One-sentence description ending with a period.
```

4. The description must start with an uppercase letter and end with a period.
5. The URL should link to the project's main page or GitHub repository.
6. Links must use full URLs (not shortlinks).
7. Only add active, maintained projects. If the project appears abandoned, please flag it in an issue first.
8. New categories require discussion via an issue before submission.

## Pull request checklist

- [ ] Entry is at the bottom of the correct category section
- [ ] Format matches: `- [Name](URL) - Description.`
- [ ] Description starts uppercase, ends with period
- [ ] No duplicates across categories
- [ ] URL is not a shortlink
- [ ] Pull request title describes the addition

## Quality standards

- Tools must be actively maintained (updated within 12 months)
- Papers must be published at a recognized venue or preprint server
- Organizations must have a public-facing presence
- Regulations must link to an official or authoritative source

## Questions

Open an issue if you are unsure about scope or placement.
